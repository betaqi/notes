## 类型提升
### 一、基本规则（Java 类型提升）

在**算术表达式中**，类型提升遵循以下顺序和规则

| 类型                    | 升级为                          |
| --------------------- | ---------------------------- |
| `byte`、`short`、`char` | 都会提升为 `int`                  |
| `int`                 | 若和 `long` 运算，则提升为 `long`     |
| `long`                | 若和 `float` 运算，则提升为 `float`   |
| `float`               | 若和 `double` 运算，则提升为 `double` |
### 二、具体规则 + 示例

#### `byte`、`short`、`char` → 自动提升为 `int`
```
byte b1 = 10;
byte b2 = 20;
// byte b3 = b1 + b2; // ❌ 错误，结果是 int，不能自动赋值给 byte
int b3 = b1 + b2;     // ✅ 正确

char c1 = 'A';
int c2 = c1 + 1;      // ✅ 正确，'A' 是 65，c2 = 66

```

#### `int` + `long` → 结果为 `long`4444
```
int i = 1000;
long l = 2000L;
long result = i + l;  // ✅ i 被提升为 long，结果是 long

```

#### `float` + `double` → 结果为 `double`
```
float f = 3.14f;
double d = 2.71;
double result = f + d; // ✅ f 被提升为 double
```

### 三、常量表达式的特殊优化（窄化）
如果参与运算的是**常量字面量（constant expressions）**，结果在目标类型范围内，**Java 编译器允许窄化赋值**：
```
final byte b1 = 10;
final byte b2 = 20;
byte sum = b1 + b2;  // ✅ 编译器优化，允许，因为是常量，结果在 byte 范围内

byte x = 10;
byte y = 20;
// byte z = x + y;    // ❌ 错误，x+y是int，不能赋值给byte
```
### 四、复合赋值运算符的“隐藏转换”

如 `+=`、`-=` 等操作符，**会自动强转回原类型**：
```
byte b = 10;
b += 1;       // ✅ 隐式强转，相当于 b = (byte)(b + 1)
b = b + 1;    // ❌ 错误，需要手动强转
```

### 五、类型提升优先级图
```
byte, short, char
        ↓
        int
        ↓
       long
        ↓
      float
        ↓
     double
```
## 二进制的 原码 反码 补码

1. 二进制的最高位是符号位：0表示正数，1表示负数（老韩口诀：0->01- -）
2. 正数的原码，反码，补码都一样（三码合一）
3. 负数的反码=它的原码符号位不变，其它位取反（0->1,1->0）
4. 负数的补码=它的反码+1，负数的反码= 负数的补码- 1
5. 0的反码，补码都是O
6. java没有无符号数，换言之，java中的数都是有符号的
7. 在计算机运算的时候，都是以补码的方式来运算的.
8. 当我们看运算结果的时候，要看他的原码（重点）

| 名称     | 含义                     |
| ------ | ---------------------- |
| **原码** | 符号位 + 数值部分（最直观）        |
| **反码** | 负数 = 原码按位取反（除了符号位）     |
| **补码** | 负数 = 反码再 +1（计算机内部实际使用） |
#### 示例

| 编码方式   |     +5      |     -5      |
| ------ | :---------: | :---------: |
| **原码** | `0000 0101` | `1000 0101` |
| **反码** | `0000 0101` | `1111 1010` |
| **补码** | `0000 0101` | `1111 1011` |

## 重载和重写

| 特性        | 重载 Overload       | 重写 Override             |
| --------- | ----------------- | ----------------------- |
| 是否跨类      | 否（同一类中）           | 是（子类重写父类的方法）            |
| 参数列表      | 不同                | 必须相同                    |
| 返回类型      | 可同可不同，不影响重载       | 可以不同，但必须是父类返回类型的子类      |
| 访问修饰符     | 无限制               | 子类权限不能低于父类              |
| static 方法 | 可以重载              | 不能重写，只能“隐藏”             |
| final 方法  | 可以重载              | 不能重写                    |
| 多态性       | 编译时多态             | 运行时多态                   |
| 注解        | 不需要 @Override（可选） | 推荐写 @Override，能检查是否正确重写 |

## *动态绑定机制*

1. ==**当调用对象方法的时候，该方法会和该对象的内存地址/运行类型绑定**==
2. 当调用对象属性时，没有动态绑定机制，哪里声明，那里使用

```
public class Index {  
    public static void main(String[] args) {  
        Parent p = new Child();  
        System.out.println(p.sum()); // 运行类型绑定 => 30    
    }  
}  
class Parent {  
    int Sum = 10;  
    public int getSum() {  
        return Sum;  
    }  
    public int sum(){  
        return this.getSum() + 10;  
    }  
}  
  
class Child extends Parent {  
    int Sum = 20;  
    @Override  
    public int getSum() {  
        return Sum;  
    }  
  
}
```
