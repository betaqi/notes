### Q1:   框选怎么判断组件在选区内
> 用的是**矩形碰撞检测**（AABB）。
> 
> mousedown 记录起始点，mousemove 实时更新一个绝对定位的选区 div 的宽高。关键是坐标换算——选区是屏幕坐标，组件是画布坐标，画布有缩放，所以选区的 left/top/width/height 都要**除以 sceneScale** 才能和组件的 x/y/w/h 做碰撞。
> 
> 碰撞判断用 `rectangleCol` 函数，本质就是判断两个矩形是否有交集：
> 
> ```
> !(rect1.right < rect2.left ||   rect1.left > rect2.right ||   rect1.bottom < rect2.top ||   rect1.top > rect2.bottom)
> ```
> 
> 有两个细节：一是做了 300ms 节流，避免 mousemove 每帧都遍历所有节点；二是只检测根层级且未锁定、未隐藏的节点，组内子节点不参与框选。

### Q2:  成组逻辑
成组就是创建一个 `type: container, group: true` 的新节点，把选中组件的 container 指向它，然后根据子节点的包围盒算出组的位置和尺寸，子节点坐标转成相对于组左上角的相对值。取消成组就是反向——子节点坐标加回组的偏移，container 指回上一级，删掉组节点。
### Q3:  成组后缩放怎么处理
> 成组后缩放用的是**等比缩放模型**。
> 
> 核心逻辑在 `groupContainerCalc`：记录组缩放前的原始宽高，缩放后算出比例 `calcX = newW / oldW`，`calcY = newH / oldH`，然后把这个比例递归应用到所有子节点的 x/y/w/h 上：
> 
> ```
> childNode.x = originalX * calcXchildNode.y = originalY * calcYchildNode.w = originalW * calcXchildNode.h = originalH * calcY
> ```
> 
> 如果子节点本身也是组，就递归调用 `groupContainerCalc`，保证嵌套组内的布局比例也跟着变。
> 
> 拖拽开始时通过 `groupContainerResizeData` 快照每个节点的原始位置尺寸，拖拽过程中基于快照算比例，避免累积误差。

证据：`node-group-methods.js:279-320`

### Q4:   定点缩放坐标变换怎么算
> 项目里的画布缩放不是数学意义上的"以某点为中心缩放"，而是用 CSS transform 实现的：
> 
> ```
> transform: scale(sceneScale) translate(canvasX, canvasY);transform-origin: left top;
> ```
> 
> `transform-origin` 固定左上角，缩放中心始终是画布左上角。要实现"以鼠标位置为中心缩放"的视觉效果，需要在改 sceneScale 的同时调整 canvasX/canvasY 偏移量，公式是：
> 
> ```
> newCanvasX = mouseX - (mouseX - oldCanvasX) * (newScale / oldScale)newCanvasY = mouseY - (mouseY - oldCanvasY) * (newScale / oldScale)
> ```
> 
> 本质是：缩放前鼠标指向画布上的某个点，缩放后这个点在屏幕上的位置不变。

另外，屏幕坐标转画布坐标的通用公式：

```
canvasCoord = (screenCoord / sceneScale) - canvasOffset
```

证据：`scene.vue:304`，`select-area.js:14`（框选里的坐标换算）
### Q5:   缩放矩阵用什么数据结构
> **没有用矩阵。** 就是三个独立数值：`sceneScale`（缩放比）、`canvasX`（X 平移）、`canvasY`（Y 平移）。
> 
> 因为变换只有缩放 + 平移，不涉及旋转和倾斜，三个变量就能完整描述画布状态。用矩阵是过度设计——矩阵的优势在于可以组合任意仿射变换（旋转、倾斜、透视），我们用不到。
> 
> 如果后续要支持组件旋转，才需要引入 2D 变换矩阵（3×3）或者用 CSS 的 matrix() 函数。
### Q6:  图层排序怎么实现
图层顺序就是**节点在数组中的索引**。Vue 的 `v-for` 按数组顺序渲染 DOM，后渲染的元素自然覆盖在前面（z-index 相同时 DOM 顺序决定层叠）。
> 
> 调整图层就是操作数组：置顶 = 移到数组末尾，置底 = 移到数组开头，上移/下移 = swap 相邻元素。
> 
> 编辑器左侧有个图层面板（`layer-item.vue`），展示所有节点的层级列表，支持拖拽排序。排序后更新数组顺序，Vue 响应式重新渲染。
### Q7:  等比缩放如何实现（用基础 width/height）
#### 场景 A：大屏预览时的等比缩放（根节点 scale） 

```
const scaleX = screenWidth / designWidth;const scaleY = screenHeight / designHeight;const scale = Math.min(scaleX, scaleY);  // 取小值保证不溢出// CSS: transform: scale(scale) translate(-50%, -50%)// 配合 position: absolute; top: 50%; left: 50% 居中
```

设计稿的 width/height 存在 root 节点的配置里，运行时拿屏幕尺寸除以设计尺寸得到缩放比。取 min 保证等比不变形，多出来的空间留黑边。

证据：`scene.vue:255-265`

####  场景 B：组内组件的等比缩放（groupContainerCalc） 

```
// 组被拖拽改变尺寸时calcX = currentGroupW / originalGroupW;  // 宽度变化比calcY = currentGroupH / originalGroupH;  // 高度变化比
// 子节点全部按比例缩放child.x = child.originalX * calcX;child.y = child.originalY * calcY;child.w = child.originalW * calcX;child.h = child.originalH * calcY;
```

用的是拖拽开始时快照的原始 w/h 作为基准（`dragSizeData`），不是每帧基于上一帧算，避免浮点累积误差。

证据：`node-group-methods.js:284-310`
证据：`editor/components/layer-item.vue`，`scene.vue:31-48`（v-for 渲染容器）
