### 一、六阶段全景图

```
阶段 0:模型生态与推理基础设施
        ↓ 能跑起来一个模型,理解协议层
阶段 1:大模型 API 与 FastAPI 后端
        ↓ 能写一个流式、容错的 AI 网关
阶段 2:工具调用与 MCP 协议
        ↓ 让模型长出手脚,接入 Cursor / Claude Code
阶段 3:企业级 RAG
        ↓ 注入私有数据,解决幻觉
阶段 4:Agent 编排与状态机
        ↓ 复杂决策、记忆、HITL、多 Agent 协作
阶段 5:商业化落地与运维
        ↓ 上线、可观测、成本可控、合规
```

每阶段是**上一阶段的升级**,而不是平行知识。阶段 0-1 是地基,2-4 是核心能力,5 是把前面一切搬上生产。

### 二、学习节奏建议

#### 三档节奏可选

|节奏|时长|适合|
|---|---|---|
|**冲刺(全职)**|8-10 周|已有 Python + 后端基础,全职转型|
|**常规(业余)**|4-6 个月|在职边学边用|
|**稳健(深耕)**|8-12 个月|兼顾理论与底层原理|

#### 各阶段时长分配(以常规节奏 4-6 个月为例)

|阶段|占比|周数|
|---|---|---|
|阶段 0|8%|1-2 周|
|阶段 1|15%|3-4 周|
|阶段 2|17%|3-4 周|
|阶段 3|22%|4-6 周|
|阶段 4|22%|4-6 周|
|阶段 5|16%|3-4 周|

**阶段 3 和 4 给最多时间**,因为这两个阶段决定你能否做出"真正有价值"的 AI 产品,其他阶段都是工具。

---

### 三、不要踩的几个坑

1. **不要在阶段 0 钻牛角尖** — 推理引擎原理可以"用到再深入",不要一开始啃 PagedAttention 论文
2. **不要跳过阶段 1 的容错** — 重试 / 限流 / 流式是体力活,但跳过后面每个阶段都补不回来
3. **不要先学框架再学原理** — 阶段 2 手写一遍 Agent Loop,再去学 LangGraph,会理解深得多
4. **不要把 RAG 做成检索玩具** — 阶段 3 必须做到带评估、带引用、带可观测,否则上不了生产
5. **不要在阶段 4 押注单一框架** — 用 LangGraph 学透原理后,至少用 OpenAI Agents SDK / PydanticAI 中的一个再写一遍
6. **阶段 5 越早接入越好** — Langfuse / 结构化日志在阶段 1 就该接,不要等到最后

### 四、贯穿六阶段的"主线项目"建议

最好选 **一个真实业务场景**,从阶段 1 开始持续迭代,而不是每阶段独立项目:

候选场景(选一个能解决你自己实际问题的):

- 个人知识库助手(笔记 / 邮件 / PDF)
- 代码库智能助手(私有仓库问答 / 代码生成)
- 客服 / 内部知识问答(如果你公司有真实场景)
- 投研 / 行业分析助手(数据多、链路长)

**好处:** 每个阶段加一层,5 个月后你有一个能上线、能演示、能写进简历的完整作品,而不是 6 个零散 demo。
