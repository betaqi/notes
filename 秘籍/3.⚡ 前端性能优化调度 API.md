浏览器提供多种异步调度方法，用于优化页面性能、避免主线程阻塞和掉帧。  

| 方法                                          | 执行时机           | 优先级 | 使用场景                       | 受主线程阻塞    |
| ------------------------------------------- | -------------- | --- | -------------------------- | --------- |
| `requestAnimationFrame`                     | 渲染前（下一帧）       | 高   | 动画、绘制、滑动、canvas 渲染         | ✅         |
| `requestIdleCallback`                       | 主线程空闲时         | 低   | 日志上报、缓存更新、非关键计算、后台任务       | ✅（仅空闲时执行） |
| `queueMicrotask`                            | 当前宏任务结束后立即执行   | 高   | Promise 回调、状态同步、轻量计算       | ✅         |
| `MutationObserver`                          | DOM 变动后异步触发微任务 | 高   | 监听 DOM 改变、布局依赖、批量 DOM 更新   | ✅         |
| `setTimeout(fn, 0)`                         | 宏任务队列          | 中   | 延迟任务、长任务切片、异步执行            | ✅         |
| `MessageChannel`                            | 宏任务队列          | 中   | 高性能替代 `setTimeout(0)`，任务切片 | ✅         |
| `ResizeObserver`                            | 元素尺寸变化时异步回调    | 高   | 响应式布局、组件自适应、监控元素大小         | ✅         |
| `IntersectionObserver`                      | 元素可视状态变化       | 高   | 图片懒加载、无限滚动、可视元素触发事件        | ✅         |
| `requestPostAnimationFrame`（自定义 / polyfill） | rAF 后宏任务       | 中   | 延迟渲染任务、切片动画                | ✅         |

---

## 🔹 使用场景总结

### 1. 动画与绘制
- **推荐 API**：`requestAnimationFrame`  
- **说明**：与浏览器刷新同步，保证动画不卡顿、流畅。

### 2. 低优先级后台任务
- **推荐 API**：`requestIdleCallback`  
- **说明**：在主线程空闲时执行，不影响动画和关键操作。

### 3. 微任务及时更新
- **推荐 API**：`queueMicrotask` / `Promise`  
- **说明**：当前宏任务完成后立即执行，用于状态同步、轻量计算。

### 4. 监听 DOM 或布局变化
- **推荐 API**：`MutationObserver`  
- **说明**：异步监听 DOM 改变，避免频繁触发 reflow / repaint。

### 5. 元素尺寸变化响应
- **推荐 API**：`ResizeObserver`  
- **说明**：用于自适应布局或组件尺寸监控。

### 6. 可视区域触发
- **推荐 API**：`IntersectionObserver`  
- **说明**：懒加载图片、无限滚动、元素可视性检测。

### 7. 长任务切片 / 异步执行
- **推荐 API**：`setTimeout(fn, 0)` / `MessageChannel` / `requestPostAnimationFrame`  
- **说明**：将耗时任务拆分成小块，避免阻塞主线程。

---

> 💡 **优化原则**：
> - 高频动画 → `requestAnimationFrame`  
> - 微任务同步 → `queueMicrotask`  
> - 主线程空闲 → `requestIdleCallback`  
> - DOM/布局观察 → `MutationObserver` / `ResizeObserver` / `IntersectionObserver`  
> - 长任务拆分 → `setTimeout` / `MessageChannel` / `requestPostAnimationFrame`
