## 💻 一、技术深度考察  
  
- **JavaScript / TypeScript 核心**  
  
  - 异步编程：事件循环（Event Loop）、微任务／宏任务顺序、`Promise` 链式调用与错误处理、`async/await` 的本质；可能要求你手写 `Promise` 或静态方法（如 `all`, `allSettled`, `race`）。  
  
  - 作用域与闭包：闭包原理、典型应用（如模块化、防抖／节流），以及闭包带来的内存泄漏问题和避免方式。  
  
  - 原型与原型链：区分 `__proto__` vs `prototype`，如何通过原型实现继承，以及 `instanceof` 的机制。  
  
  - ES6+ 特性：箭头函数 vs 普通函数，`let`／`const` 的块级作用域与暂时性死区 (TDZ)，`Symbol`, `BigInt` 的应用场景，解构赋值，模块化 (ES Module) 等。  
  
  - TypeScript：基础类型、接口 (Interface)、泛型 (Generics)、类型守卫 (Type Guards) 等优点 —— 并理解为什么在项目中会使用 TS。  
  
- **框架原理 (React / Vue)**  
  
  - **React**  
  
      - 理解 Fiber 架构：为什么 React 可以实现异步、可中断更新，以及协调 (Reconciliation) 过程。  
  
      - Hooks 原理：常用 Hooks（如 `useState`, `useEffect`, `useMemo`, `useCallback`）的内部机制和使用场景; `useEffect` 的依赖数组与清理机制；为什么不能将 Hooks 放在条件语句中。  
  
      - 状态管理：理解 `setState` 的同步/异步行为，以及 Redux / MobX 等状态管理库的设计思想和使用（即使公司内部可能使用自研方案）。  
  
      - 性能优化：例如 `React.memo`, `useMemo`, `useCallback`，组件懒加载 (`React.lazy` + `Suspense`) 等。  
  
  - **Vue**  
  
      - 响应式原理：Vue 2 用 `Object.defineProperty` vs Vue 3 用 `Proxy`，了解各自的优劣。  
  
      - Composition API vs Options API：`ref`, `reactive` 的使用，以及 `setup` 中生命周期钩子与 Options API 的对应关系。  
  
  - **框架对比**：可能被问到对 React 与 Vue 的理解与比较，准备好阐述自己的看法。  
  
- **浏览器与网络**  
  
  - 渲染机制：从发送 URL 开始，到页面展示完成，经历 DNS 解析、TCP 连接、HTTP 请求 → HTML 解析 → 构建 CSSOM → 构建渲染树 → 布局(Layout) → 绘制(Paint) 等流程。  
  
  - 缓存策略：理解强缓存 (`Cache-Control`, `Expires`) 与协商缓存 (`ETag`, `Last-Modified`) 的区别与流程 — 包括 304 请求是否会发送给服务器。  
  
  - HTTP 版本：对比 HTTP/1.1、HTTP/2、HTTP/3 各自主要特性，如 HTTP/2 的多路复用与头部压缩，HTTP/3 的 QUIC 协议。  
  
  - 跨域 (CORS)：为什么会跨域，常见解决方案 (CORS、JSONP、Nginx 反向代理等) 。  
  
- **性能优化**  
  
  - 加载性能优化：减少 HTTP 请求数、压缩资源、利用浏览器缓存、代码拆分 / 分割 (code-splitting)、懒加载 (lazy loading)、预加载 (preload)、启用 Gzip、使用 CDN 等。  
  
  - 运行时性能优化：避免频繁操作 DOM、防抖 (debounce) / 节流 (throttle)、优化重绘 (repaint) 与回流 (reflow)、使用 Web Worker 处理耗时任务、使用虚拟列表 (virtual list) 优化长列表渲染。  
  
  - 工程化优化：Tree Shaking（移除未使用代码）、按需加载第三方库等。  
  
  - 性能监控与指标：了解 Web Vitals (如 LCP, FID, CLS) 并知道对应优化手段。  
  
- **工程化与工具**  
  
  - 构建工具：了解打包工具（如 Webpack）的打包流程、Loader / Plugin 的作用 & 优化技巧 (code splitting, lazy loading)；了解新工具 (如 Vite) 的机制 (如 ESBuild 预编译、热模块替换 HMR) 以及其与传统打包工具的区别。  
  
  - 代码规范：了解 ESLint、Prettier、Husky 等在项目中的作用。  
  
  - 版本控制 / 协作工具：熟悉 Git 常见工作流、合并 (merge) 与冲突解决等。  
  
- **前端安全**  
  
  - XSS（跨站脚本攻击）：了解 XSS 的原理、危害及防范 (例如输入转义、CSP) 。  
  
  - CSRF（跨站请求伪造）：了解其原理、危害及防范方式 (例如 SameSite Cookie、Token 验证) 。

## 📂 二、项目经验深挖  
  
- 面试官可能围绕你简历中的项目 **深入提问** ——  
  
  - 要清晰介绍你负责项目的背景 / 目标 /主要模块 /技术选型原因。  
  
  - 说明你在项目中遇到的具体**技术难点**，以及你是如何解决的 (比如性能瓶颈、复杂业务逻辑、兼容性问题等) 。  
  
  - 如果可能，准备复盘 (如果重新做该项目，你会如何优化) — 包括稳定性、监控 / 报警机制等。  
  
  - 对项目质量保障 (如内存泄漏判断与定位, 日常巡检流程, 大页面白屏检测 / API 异常监控) 有自己的理解与方案。  
  
  
建议使用 **STAR 法则** (Situation → Task → Action → Result) 来结构化项目描述。准备 1–2 个能够体现你技术深度和问题解决能力的项目，以备面试深入追问。

## 💻 四、编码能力考察  
  
三面可能还会包含实际编码题 — 更贴近实际业务 / 综合考察能力，可能包括：  
  
- 手写常见功能，例如防抖 (debounce)、节流 (throttle)、深拷贝 (特别要注意循环引用)、实现 `Promise`、实现 `call`/`apply`/`bind`、实现数组扁平化 (`flatMap`) 等。  
  
- 算法与数据结构相关：可能考查中等难度的算法，如排序 (快速排序、归并排序)、二叉树操作、字符串处理 (例如 “第一个不重复字符” 或其他字符串题)、数组操作 (比如找数组中前 n 个最大 / 最小元素、合并有序数组) 等。也可能考一些设计题 (例如设计一个队列、设计一个弹幕系统等)。

## 💡 五、面试技巧与备战建议  
  
1. **自我介绍 + 项目陈述**：准备清晰、简洁、有亮点的自我介绍。介绍项目时，突出你的**个人贡献、技术决策过程**以及带来的价值。  
  
2. 对技术 / 特性 / 优化方案，不仅要“知道是什么 (what)”，更要“知道为什么 (why)”和“如何应用 / 实现 (how)”。例如性能优化，要说明为什么能优化、背后的原理是什么。  
  
3. 如果遇到不会的问题，诚实 + 展示自己的 **思考能力 / 学习能力**。与面试官沟通，把面试当成交流，而不是死记答案。  
  
4. **反问环节**：要提前准备一些富有深度的问题，例如团队的业务方向、面临的技术挑战、团队技术氛围、新人培养机制等 — 展示你对公司 / 职位的关注与思考。
