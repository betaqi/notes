# 防抖
#### 标准通用版 

1. **`this` 指向丢失问题**：在 `setTimeout` 中执行 `fn` 时，`this` 默认指向 `window` (非严格模式) 或 `undefined`。我们需要它指向调用者（比如 input 元素）。
2. **参数传递问题**：事件对象 `event` 或其他参数需要透传给 `fn`。


```
/**
 * @param {Function} fn - 需要防抖的函数
 * @param {Number} delay - 延迟时间 (ms)
 */
function debounce(fn, delay) {
  let timer = null;

  // 返回的函数是用户实际调用的函数（例如绑定在 input 上的回调）
  return function(...args) { 
    const context = this; // 1. 保存当前作用域的 this (例如 input DOM 元素)

    if (timer) clearTimeout(timer); // 清除上一次的定时器

    timer = setTimeout(() => {
      // 2. 利用 apply 修正 this 指向，并传递参数
      fn.apply(context, args); 
    }, delay);
  }
}
```
##### 3. 常见问题

**Q1: 为什么要用 `fn.apply(context, args)` 而不是直接 `fn(args)`？**

> **答**：为了保证函数执行时的上下文正确。 假设我们在 input 上绑定了防抖函数：`<input oninput="debouncedSearch()">`。 原本的 `search` 函数内部如果使用了 `this.value`，直接调用 `fn()` 会导致 `this` 变成 `window`，拿不到 input 的值。必须用 `apply` 把 `debounce` 返回函数接收到的 `this` （即 input 元素）传递给 `fn`。

#### 进阶版 (支持“立即执行”)

**场景**：用户点击按钮，希望**立刻**触发一次，然后后续的连续点击才会被忽略（防抖）。 **面试加分项**：增加 `immediate` 参数。

```
function debounce(fn, delay, immediate = false) {
  let timer = null;

  return function(...args) {
    const context = this;

    if (timer) clearTimeout(timer);

    if (immediate) {
      // 如果 timer 为 null，说明是第一次点击（或上一个周期结束）
      const callNow = !timer; 
      
      // 设置定时器，仅仅是为了在 delay 之后重置 timer 为 null，
      // 以便允许下一次“立即执行”
      timer = setTimeout(() => {
        timer = null;
      }, delay);

      // 如果符合条件，立即执行
      if (callNow) fn.apply(context, args);
      
    } else {
      // 标准的防抖逻辑
      timer = setTimeout(() => {
        fn.apply(context, args);
      }, delay);
    }
  }
}
```


# 截流
### 3. 高级进阶版 (Lodash 风格)

**面试官追问**：“我想要刚开始触发时立刻执行一次（时间戳的好处），最后停止触发时还能再执行一次（定时器的好处），怎么做？”
```
function throttle(fn, delay) {
  let timer = null;
  let startTime = 0;

  return function(...args) {
    const now = Date.now();
    const remaining = delay - (now - startTime); // 距离下次执行还剩多久
    const context = this;

    clearTimeout(timer);

    // 1. 如果没有剩余时间了（即已经超过了 delay），或者系统时间被改了（now < startTime）
    // 这里的逻辑主要对应“立即执行”和“时间间隔到了”
    if (remaining <= 0) {
      fn.apply(context, args);
      startTime = now;
    } else {
      // 2. 如果还在时间内，则设立定时器，保证最后一次触发能被执行（Trailing）
      timer = setTimeout(() => {
        fn.apply(context, args);
        startTime = Date.now(); // 注意：这里要更新 startTime，防止下一次立即触发
      }, remaining);
    }
  }
}
```

### 4.  (防抖 vs 节流)

|**特性**|**防抖 (Debounce)**|**节流 (Throttle)**|
|---|---|---|
|**执行时机**|只有当你**停下来**不再触发时才执行|按照**固定的频率**规律执行|
|**主要应用**|搜索框联想 (`input`)、窗口大小调整结束 (`resize`)|页面滚动 (`scroll`)、鼠标移动 (`mousemove`)、视频播放进度保存|
|**直观感受**|延迟感强，不操作才有反馈|有规律的跳动感，反馈由于延迟|

# apply
**核心思路**：与 `call` 几乎一样，唯一的区别是处理第二个参数（必须是数组或类数组）。

JavaScript

```
Function.prototype.myApply = function(context, args) {
  if (typeof this !== 'function') {
    throw new TypeError('Error');
  }
  context = context || window;
  
  const fnSymbol = Symbol();
  context[fnSymbol] = this;

  let result;
  // 判断 args 是否存在，如果存在则展开数组
  if (args) {
    // 简单判断是否是数组/类数组
    if (!Array.isArray(args) && !args.length) { 
        throw new TypeError('CreateListFromArrayLike called on non-object'); 
    }
    result = context[fnSymbol](...args);
  } else {
    result = context[fnSymbol]();
  }

  delete context[fnSymbol];
  return result;
};
```
